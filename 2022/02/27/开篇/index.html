开篇

很久之前就想要找个地方记录下自己写的东西，但是一直拖延着没有去做。今天突然有了兴致就索性先做起来。

这是第一篇，也不知道写什么。就记录一下，这个 blog 建立的过程吧。

1. Hexo

选择 Hexo 作为静态博客工具主要是因为它是使用Node构建的，不需要额外安装其他语言工具。

按照 Hexo 官网的文档，首先配置电脑环境。需要的软件也不多，只要有 Node.js 和 git。然后使用 npm 全局安装 Hexo。npm install -g hexo-cli

2. 从无到有

安装完Hexo，运行hexo init blog，Hexo 程序会自动创建一个 blog 文件夹。其中有很多文件，先不管这些。我只想尽快开始写作并发布。

首先修改_config.yml 文件。这是博客的配置文件，比如设置博客的标题，就修改 title 属性，还有作者名字title, 语言language, 时区timezone等等。更多的配置可以在官网文档中找到。

这里提一下root属性。由于之后我是用Github Pages 部署blog，github会在访问地址中加入仓库名。比如我的访问首页地址是https://hkeimo.github.io/blog，需要设置root 为 /blog/

完成配置就可以正式开始写了。运行hexo new <title>，Hexo 在 source/_posts/ 下会新建 md 文件，在这里就可以写了。写完后运行 hexo generate 生成public 文件夹，这里包含了blog所有的前端文件。接下去就是怎么把这个文件夹的内容发布的问题了。

3. Hello world

Github 提供 Github Pages 提供静态文件网站的部署，正好符合我的需求。

在Github 新建一个公开的仓库，把上面生成的public 文件夹中内容推送到这个仓库中，然后开启仓库的Github Pages功能。很简单吧！

到此就可以了。等待部署完成，打开 Github Pages 提供的链接就可以看到blog了。

4.最后

祝贺自己大功告成。完成一件事也许不需要花费很多时间，但是往往拖延着。不去做，久而久之就会不想去了做了。这大概是人性的一部分吧。生活中的诱惑很多，而真正有价值的却很少。从这里开始，希望自己以后能珍惜宝贵的时间，保持专注。
